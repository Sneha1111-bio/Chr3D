### **🚨 ISSUE 1: WRONG Default for `keep_dup`**


**PROBLEM:** Conflicting defaults! Class init uses `'1'` (remove duplicates), but CLI uses `'all'` (keep duplicates).

**ChIA-PET Best Practice:**[3][4][1][2]
- **Self-ligation PETs (sPETs)**: Use `--keep-dup all` ✅
  - sPETs represent **real binding events** (not PCR artifacts)
  - Each sPET is a unique molecule from the experiment
  - Removing duplicates **loses biological signal**

- **Regular ChIP-seq**: Use `--keep-dup 1` or `auto`
  - PCR duplicates are technical artifacts
  - Should be removed

**Your CLI default (`'all'`) is CORRECT for ChIA-PET**, but the class default should match:

```python
def __init__(self, 
             keep_dup: str = 'all',  # ✅ CORRECT for ChIA-PET
             ...):
```

### **🚨 ISSUE 2: Force `--nomodel` is TOO AGGRESSIVE**

```python
# Add model building options
# For ChIA-PET data with single-base anchors, always use --nomodel
if not self.build_model or True:  # Force nomodel for ChIA-PET
    cmd.extend(['--nomodel', '--extsize', '200'])
```

**PROBLEM:** The `or True` makes `build_model` parameter USELESS.[5][6][7]

**When to use `--nomodel`:**[6][8][5]
1. **Single-end ChIA-PET data**: YES, use `--nomodel --extsize 200`
   - Tags are single bases, no strand shift to model
   - Extension length = sonication fragment size (~200bp typical)

2. **Paired-end ChIA-PET with BAM**: NO, let MACS3 build model
   - MACS3 can learn fragment size distribution from actual inserts
   - More accurate than fixed 200bp extension

3. **BEDPE format**: Depends on data
   - If BEDPE already contains fragment coordinates → use `BEDPE` format (MACS3 uses actual coordinates)
   - If BEDPE has single-base anchors → use `BED` with `--nomodel`

**Your current code ALWAYS uses `--nomodel` regardless of data type!**

**Recommended fix:**

```python
# Determine if nomodel is appropriate
use_nomodel = not self.build_model

# For single-base anchor data (BED from BEDPE conversion), force nomodel
if format == 'BED' and not input_file.endswith('.bam'):
    use_nomodel = True
    logger.info("  Using --nomodel for single-base anchor data")

    cmd.extend(['--nomodel', '--extsize', '200'])
    logger.info(f"  Extension size: 200bp (fixed)")

    logger.info("  Building MACS3 shift model from data")
```

**PROBLEM:** When you pass `format='BEDPE'`, MACS3 expects **3-column BEDPE** format:[9][5]

But your BEDPE files have **10-11 columns** (standard BEDPE):

**MACS3 will FAIL or misinterpret this!**[9]

**Two solutions:**

**Option A: Convert to MACS3 BEDPE format (3-column)**

```python
def bedpe_to_macs3_bedpe(self, bedpe_file: str, output_file: str) -> str:
    """
    Convert standard BEDPE to MACS3 3-column BEDPE format.
    
    MACS3 BEDPE format: chr, leftmost_pos, rightmost_pos
    This treats each PET as a single fragment spanning both anchors.
    """
    df = pd.read_csv(bedpe_file, sep='\t', header=None, comment='#',
                    names=['chr1', 'start1', 'end1', 'chr2', 'start2', 'end2',
                          'name', 'score', 'strand1', 'strand2', 'weight'])
    df = df.dropna()
    
    # Only keep intra-chromosomal PETs (same chromosome)
    df = df[df['chr1'] == df['chr2']]
    
    # Create MACS3 BEDPE: chr, min(start1, start2), max(end1, end2)
    macs3_bedpe = pd.DataFrame({
        'chr': df['chr1'],
        'start': df[['start1', 'start2']].min(axis=1),
        'end': df[['end1', 'end2']].max(axis=1)
    })
    
    macs3_bedpe.to_csv(output_file, sep='\t', header=False, index=False)
    
    logger.info(f"  Converted {len(df)} intra-chr PETs to MACS3 BEDPE")
    logger.info(f"  (Dropped {(df['chr1'] != df['chr2']).sum()} inter-chr PETs)")
    
    return output_file
```

**Option B: Remove BEDPE method entirely** (your current BED conversion is better for ChIA-PET)

I recommend **Option B** - the BED conversion approach is the **ChIA-PET standard**.[1][2]
