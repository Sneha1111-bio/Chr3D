# Hi-C Mode CLI

Hi-C mode analyzes genome-wide chromatin architecture using proximity ligation sequencing. This mode generates contact matrices at multiple resolutions using pairtools and cooler.

## Overview

Hi-C captures all chromatin interactions genome-wide without protein-specific enrichment. The Chr3D pipeline:

1. **Genomic Mapping** - Maps paired-end reads using BWA-MEM
2. **Pairtools Processing** - Parses, sorts, deduplicates, and filters pairs
3. **Cooler Matrix Generation** - Creates multi-resolution contact matrices (.mcool)
4. **QC Statistics** - Generates comprehensive quality metrics

**Note**: Hi-C mode does **not** include peak calling or loop calling (no protein enrichment).

## Command

```bash
chr3d run --mode hic [options]
```

## Required Arguments

| Argument | Description |
|----------|-------------|
| `--fastq-r1` | Input R1 FASTQ file (gzipped supported) |
| `--fastq-r2` | Input R2 FASTQ file (gzipped supported) |
| `--genome-index` | BWA-indexed reference genome FASTA |
| `--chrom-sizes` | Chromosome sizes file (required for Hi-C) |
| `--output-dir` | Output directory |

## Optional Arguments

| Argument | Default | Description |
|----------|---------|-------------|
| `--threads` | 24 | Number of threads |
| `--mapping-quality` | 30 | Minimum mapping quality |
| `--min-distance` | 1000 | Minimum distance for valid pairs (bp) |
| `--resolution` | `1000,5000,10000` | Resolutions for contact matrices (comma-separated) |
| `--assembly` | `hg38` | Genome assembly name |
| `--sample-id` | `sample` | Sample identifier |
| `--keep-intermediates` | False | Keep intermediate files |

## Complete Example

```bash
chr3d run --mode hic \
    --fastq-r1 /data/HiC_sample_R1.fastq.gz \
    --fastq-r2 /data/HiC_sample_R2.fastq.gz \
    --genome-index /ref/hg38.fa \
    --chrom-sizes /ref/hg38.chrom.sizes \
    --output-dir /output/hic_analysis \
    --threads 16 \
    --mapping-quality 30 \
    --min-distance 1000 \
    --resolution 1000,5000,10000,25000,50000,100000 \
    --assembly hg38 \
    --sample-id HiC_experiment \
    --keep-intermediates
```

## Generating Chromosome Sizes File

If you don't have a chromosome sizes file:

```bash
# Using samtools
samtools faidx /ref/hg38.fa
cut -f1,2 /ref/hg38.fa.fai > /ref/hg38.chrom.sizes

# Or download from UCSC
wget https://hgdownload.soe.ucsc.edu/goldenPath/hg38/bigZips/hg38.chrom.sizes
```

## Pipeline Steps

### Step 1: BWA-MEM Alignment

Maps paired-end reads to reference genome.

```
Input:  sample_R1.fastq.gz, sample_R2.fastq.gz
Output: aligned/sample.bam
```

**Alignment Statistics:**
- Total read pairs
- Mapped reads
- Mapping quality distribution

### Step 2: Pairtools Parse

Converts aligned BAM to pairs format with interaction classification.

```
Input:  aligned/sample.bam
Output: pairs/sample.pairs.gz
```

**Pair Types:**
- `UU`: Both reads uniquely mapped
- `UR/RU`: One unique, one rescued
- `MU/UM`: One multi-mapped
- `MM`: Both multi-mapped
- `NN`: Both unmapped

### Step 3: Pairtools Sort

Sorts pairs by genomic coordinates.

```
Input:  pairs/sample.pairs.gz
Output: pairs/sample.sorted.pairs.gz
```

### Step 4: Pairtools Dedup

Removes PCR duplicates.

```
Input:  pairs/sample.sorted.pairs.gz
Output: pairs/sample.dedup.pairs.gz
```

**Deduplication Statistics:**
- Total pairs
- Unique pairs
- Duplicate rate

### Step 5: Pairtools Select

Filters for valid cis pairs above minimum distance.

```
Input:  pairs/sample.dedup.pairs.gz
Output: pairs/sample.filtered.pairs.gz
```

**Filtering Criteria:**
- Cis pairs only (same chromosome)
- Distance ≥ `--min-distance`
- Mapping quality ≥ `--mapping-quality`

### Step 6: Cooler Matrix Generation

Creates multi-resolution contact matrices.

```
Input:  pairs/sample.filtered.pairs.gz
Output: matrices/sample.mcool
```

**Resolutions:**
- Default: 1kb, 5kb, 10kb
- Custom: Use `--resolution` flag

## Output Formats

### Pairs Format (.pairs.gz)

Standard 4DN pairs format:

```
## pairs format v1.0
#columns: readID chr1 pos1 chr2 pos2 strand1 strand2 pair_type
read1   chr1    1000    chr1    5000    +   -   UU
read2   chr1    2000    chr2    3000    +   +   UU
```

### Cooler Format (.mcool)

Multi-resolution cooler file containing:
- Contact matrices at each resolution
- Bin annotations
- Chromosome information

**Accessing with Python:**
```python
import cooler

# Load specific resolution
clr = cooler.Cooler("sample.mcool::resolutions/10000")

# Get matrix for region
matrix = clr.matrix(balance=False).fetch("chr1:1000000-2000000")
```

### QC Statistics

The `qc/sample_pairs.stats` file contains:

| Metric | Description |
|--------|-------------|
| total | Total read pairs |
| total_mapped | Successfully mapped pairs |
| total_dups | Duplicate pairs |
| total_nodups | Unique pairs |
| cis | Same-chromosome pairs |
| trans | Different-chromosome pairs |
| cis_1kb+ | Cis pairs &gt;1kb apart |
| cis_10kb+ | Cis pairs &gt;10kb apart |
| summary/frac_cis | Fraction of cis pairs |

## Quality Metrics

**Good Hi-C Data:**

| Metric | Good | Acceptable | Poor |
|--------|------|------------|------|
| Mapping rate | &gt;90% | 70-90% | &lt;70% |
| Cis ratio | &gt;60% | 40-60% | &lt;40% |
| Cis &gt;10kb | &gt;40% | 20-40% | &lt;20% |
| Duplicate rate | &lt;20% | 20-40% | &gt;40% |

**Interpreting Results:**
- **High trans ratio**: May indicate random ligation or contamination
- **Low cis &gt;10kb**: Short-range bias, may need optimization
- **High duplicate rate**: Low library complexity

## Resolution Guidelines

| Resolution | Use Case | Memory |
|------------|----------|--------|
| 1kb | Fine-scale loops | High |
| 5kb | Loop detection | Medium |
| 10kb | TAD calling | Medium |
| 25kb | Compartments | Low |
| 50kb | Overview | Low |
| 100kb | Whole-genome view | Very Low |

**Recommendations:**
- Loop calling: 5-10kb
- TAD calling: 10-25kb
- Compartment analysis: 25-100kb
- Visualization: Multiple resolutions

## Downstream Analysis

### With Cooler/Cooltools

```bash
# Balance matrix (ICE normalization)
cooler balance sample.mcool::resolutions/10000

# Call TADs
cooltools insulation sample.mcool::resolutions/10000 \
    --window-pixels 10 \
    -o sample_insulation.tsv

# Call compartments
cooltools eigs-cis sample.mcool::resolutions/100000 \
    --phasing-track gc_content.bw \
    -o sample_compartments
```

### With HiCExplorer

```bash
# Convert to h5 format
hicConvertFormat -m sample.mcool::resolutions/10000 \
    -o sample.h5 --inputFormat cool --outputFormat h5

# Find TADs
hicFindTADs -m sample.h5 \
    --outPrefix sample_tads \
    --correctForMultipleTesting fdr
```

### With Juicer Tools

```bash
# Convert to .hic format
java -jar juicer_tools.jar pre \
    sample.filtered.pairs.gz \
    sample.hic \
    hg38.chrom.sizes
```

## Hi-C vs HiChIP vs ChIA-PET

| Feature | Hi-C | HiChIP | ChIA-PET |
|---------|------|--------|----------|
| Enrichment | None | ChIP | ChIP |
| Interactions | All | Protein-specific | Protein-specific |
| Resolution | Genome-wide | Focused | Focused |
| Input cells | 1-10M | 1M | 100M |
| Loop calling | External tools | Built-in | Built-in |
| Peak calling | No | Yes | Yes |

## Tips

1. **Sequencing Depth**: Aim for &gt;500M read pairs for 1kb resolution
2. **Chromosome Sizes**: Must match your reference genome exactly
3. **Memory**: High resolutions require significant RAM (&gt;64GB for 1kb)
4. **Storage**: .mcool files can be large (10-100GB)
5. **Normalization**: Use `cooler balance` for downstream analysis

## Troubleshooting

### Low cis ratio
- Check for contamination
- Verify ligation efficiency
- May indicate protocol issues

### High duplicate rate
- Increase library complexity
- Sequence deeper
- Optimize PCR cycles

### Memory errors
- Reduce resolution
- Increase available RAM
- Process chromosomes separately

### Cooler errors
- Verify chromosome sizes file format
- Check pairs file integrity
- Ensure sufficient disk space
