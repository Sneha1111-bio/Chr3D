# ChIA-PET Mode CLI

ChIA-PET (Chromatin Interaction Analysis by Paired-End Tag sequencing) mode analyzes protein-mediated chromatin interactions using linker-based paired-end sequencing.

## Overview

ChIA-PET uses **bridge linkers** (biotinylated synthetic DNA sequences) during the ligation step. The Chr3D pipeline:

1. **Linker Filtering** - Identifies and removes linker sequences from reads
2. **Genomic Mapping** - Maps filtered reads to reference genome using BWA
3. **PET Purifying** - Removes duplicates and merges overlapping PETs
4. **PET Categorization** - Classifies into iPET (inter-ligation), sPET (self-ligation), oPET (other)
5. **Peak Calling** - Calls peaks from sPET using MACS3
6. **Loop Calling** - Identifies significant chromatin loops with FDR correction

## Command

```bash
chr3d run --mode chiapet [options]
```

## Required Arguments

| Argument | Description |
|----------|-------------|
| `--fastq-r1` | Input R1 FASTQ file (gzipped supported) |
| `--fastq-r2` | Input R2 FASTQ file (gzipped supported) |
| `--genome-index` | BWA-indexed reference genome FASTA |
| `--linker-a` | Primary linker sequence (required for ChIA-PET) |
| `--output-dir` | Output directory |

## Optional Arguments

| Argument | Default | Description |
|----------|---------|-------------|
| `--linker-b` | None | Secondary linker sequence (for dual-linker protocols) |
| `--threads` | 24 | Number of threads |
| `--use-bwa-mem` | False | Use BWA-MEM instead of BWA-ALN |
| `--genome-size` | `hs` | Genome size for MACS3 (`hs`, `mm`, or number) |
| `--mapping-quality` | 30 | Minimum mapping quality |
| `--self-ligation-cutoff` | 8000 | Self-ligation distance cutoff (bp) |
| `--extension-length` | 500 | Tag extension length for clustering (bp) |
| `--ipet-threshold` | 2 | Minimum iPET count for loops |
| `--fdr-cutoff` | 0.05 | FDR cutoff for significant loops |
| `--sample-id` | `sample` | Sample identifier |
| `--keep-intermediates` | False | Keep intermediate files |
| `--keep-sam` | False | Keep SAM files |
| `--no-dedup` | False | Skip PCR duplicate removal |

## Complete Example

```bash
chr3d run --mode chiapet \
    --fastq-r1 /data/sample_R1.fastq.gz \
    --fastq-r2 /data/sample_R2.fastq.gz \
    --genome-index /ref/hg38.fa \
    --linker-a ACGCGATATCTTATCTGACT \
    --linker-b AGTCAGATAAGATATCGCGT \
    --output-dir /output/chiapet_analysis \
    --threads 16 \
    --use-bwa-mem \
    --genome-size hs \
    --mapping-quality 30 \
    --self-ligation-cutoff 8000 \
    --extension-length 500 \
    --ipet-threshold 2 \
    --fdr-cutoff 0.05 \
    --sample-id CTCF_ChIAPET \
    --keep-intermediates
```

## Pipeline Steps

### Step 1: Linker Filtering

Identifies linker sequences in reads using Smith-Waterman alignment and extracts genomic tags.

```
Input:  sample_R1.fastq.gz, sample_R2.fastq.gz
Output: step1_linker_filtering/filtered.*.R1.fastq
        step1_linker_filtering/filtered.*.R2.fastq
```

**Linker Types:**
- **1-1**: Both reads have linker A
- **2-2**: Both reads have linker B  
- **1-2/2-1**: Chimeric (one of each)

### Step 2: Genomic Mapping

Maps linker-filtered reads to reference genome.

```
Input:  Filtered FASTQ files
Output: step2_mapping/mapped.dedup.bedpe
```

**Mapping Statistics:**
- Total read pairs
- Mapped pairs
- Unique PETs (after deduplication)
- Intra/inter-chromosomal ratios

### Step 3: PET Purifying

Removes duplicates and merges overlapping PETs.

```
Input:  step2_mapping/mapped.dedup.bedpe
Output: step3_purifying/purified.merged.bedpe
```

### Step 4: PET Categorization

Classifies PETs based on genomic distance and orientation.

```
Input:  step3_purifying/purified.merged.bedpe
Output: step4_categorization/categorized.ipet  (inter-ligation PETs)
        step4_categorization/categorized.spet  (self-ligation PETs)
        step4_categorization/categorized.opet  (other PETs)
```

**PET Categories:**
- **iPET (Inter-ligation)**: Long-range interactions (distance > cutoff)
- **sPET (Self-ligation)**: Short-range/same-fragment (distance ≤ cutoff)
- **oPET (Other)**: Wrong orientation or unmapped

### Step 5: Peak Calling

Calls peaks from self-ligation PETs using MACS3.

```
Input:  step4_categorization/categorized.spet
Output: step5_peaks/peaks_peaks.narrowPeak
        step5_peaks/peaks_summits.bed
```

### Step 6: Loop Calling

Identifies significant chromatin loops with statistical testing.

```
Input:  step4_categorization/categorized.ipet
Output: step6_loops/loops.cluster.FDRfiltered.txt
```

**Sub-steps:**
1. **Pre-clustering**: Extends tags and creates initial clusters
2. **Anchor clustering**: Merges overlapping clusters into anchors
3. **Statistical significance**: Poisson-based test with FDR correction

## Output Format

### Loop File Format (BEDPE)

The final loop file (`loops.cluster.FDRfiltered.txt`) contains:

| Column | Description |
|--------|-------------|
| chr1 | Chromosome of anchor 1 |
| start1 | Start position of anchor 1 |
| end1 | End position of anchor 1 |
| chr2 | Chromosome of anchor 2 |
| start2 | Start position of anchor 2 |
| end2 | End position of anchor 2 |
| iPET_count | Number of supporting iPETs |
| expected | Expected count (background) |
| distance | Genomic distance |
| p_value | Raw p-value |
| FDR | Benjamini-Hochberg adjusted p-value |

## Common Linker Sequences

| Protocol | Linker A | Linker B |
|----------|----------|----------|
| ChIA-PET Tool | `ACGCGATATCTTATCTGACT` | `AGTCAGATAAGATATCGCGT` |
| ENCODE | `GTTGGATAAG` | `GTTGGAATGT` |

## Tips

1. **Linker Quality**: Ensure linker sequences match your protocol exactly
2. **Mapping Quality**: Use `--mapping-quality 30` for high-confidence interactions
3. **Self-ligation Cutoff**: Adjust based on your protein's expected interaction range
4. **FDR Threshold**: Use 0.05 for discovery, 0.01 for high-confidence loops
5. **Memory**: For large datasets, ensure sufficient RAM (>32GB recommended)

## Troubleshooting

### Low linker detection rate
- Verify linker sequences match your protocol
- Check FASTQ quality scores
- Try lowering alignment score threshold

### Few significant loops
- Increase sequencing depth
- Check peak calling results (need peaks for enrichment)
- Verify protein binding (ChIP-seq validation)

### High duplicate rate
- May indicate low library complexity
- Consider deeper sequencing or library optimization
