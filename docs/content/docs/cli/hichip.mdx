# HiChIP Mode CLI

HiChIP (Hi-C + ChIP) mode analyzes protein-centric chromatin interactions using restriction enzyme-based proximity ligation with immunoprecipitation enrichment.

## Overview

**Important**: HiChIP does **NOT** use linker sequences. Unlike ChIA-PET, HiChIP uses biotinylated nucleotides incorporated during fill-in after restriction digestion. The Chr3D pipeline:

1. **Genomic Mapping** - Maps reads directly to reference genome (no linker filtering)
2. **PET Purifying** - Removes same-fragment PETs using restriction site information
3. **PET Categorization** - Classifies into iPET, sPET, oPET based on distance
4. **Peak Calling** - Calls peaks from sPET using MACS3
5. **Loop Calling** - Identifies significant loops with distance-corrected statistics

## Command

```bash
chr3d run --mode hichip [options]
```

## Required Arguments

| Argument | Description |
|----------|-------------|
| `--fastq-r1` | Input R1 FASTQ file (gzipped supported) |
| `--fastq-r2` | Input R2 FASTQ file (gzipped supported) |
| `--genome-index` | BWA-indexed reference genome FASTA |
| `--restriction-sites` | Restriction sites BED file (required for HiChIP) |
| `--output-dir` | Output directory |

## Optional Arguments

| Argument | Default | Description |
|----------|---------|-------------|
| `--threads` | 24 | Number of threads |
| `--use-bwa-mem` | False | Use BWA-MEM instead of BWA-ALN |
| `--genome-size` | `hs` | Genome size for MACS3 (`hs`, `mm`, or number) |
| `--mapping-quality` | 30 | Minimum mapping quality |
| `--self-ligation-cutoff` | 8000 | Self-ligation distance cutoff (bp) |
| `--extension-length` | 500 | Tag extension length for clustering (bp) |
| `--ipet-threshold` | 2 | Minimum iPET count for loops |
| `--fdr-cutoff` | 0.05 | FDR cutoff for significant loops |
| `--sample-id` | `sample` | Sample identifier |
| `--keep-intermediates` | False | Keep intermediate files |
| `--keep-sam` | False | Keep SAM files |
| `--no-dedup` | False | Skip PCR duplicate removal |

## Generating Restriction Sites

Before running HiChIP analysis, generate the restriction sites file:

```bash
chr3d generate-sites \
    --genome /path/to/hg38.fa \
    --enzyme MboI \
    --output /path/to/hg38_MboI_sites.bed
```

**Common Restriction Enzymes:**

| Enzyme | Recognition Site | Used In |
|--------|------------------|---------|
| MboI | GATC | Most HiChIP protocols |
| DpnII | GATC | Alternative to MboI |
| HindIII | AAGCTT | Some Hi-C protocols |
| NcoI | CCATGG | Specific applications |

## Complete Example

```bash
# Step 1: Generate restriction sites (one-time)
chr3d generate-sites \
    --genome /ref/hg38.fa \
    --enzyme MboI \
    --output /ref/hg38_MboI_sites.bed

# Step 2: Run HiChIP pipeline
chr3d run --mode hichip \
    --fastq-r1 /data/H3K27ac_HiChIP_R1.fastq.gz \
    --fastq-r2 /data/H3K27ac_HiChIP_R2.fastq.gz \
    --genome-index /ref/hg38.fa \
    --restriction-sites /ref/hg38_MboI_sites.bed \
    --output-dir /output/hichip_analysis \
    --threads 16 \
    --use-bwa-mem \
    --genome-size hs \
    --mapping-quality 30 \
    --extension-length 500 \
    --ipet-threshold 2 \
    --fdr-cutoff 0.05 \
    --sample-id H3K27ac_HiChIP
```

## Pipeline Steps

### Step 1: Skipped (No Linker Filtering)

HiChIP does not use linkers, so this step is automatically skipped.

```
Log output: "STEP 1: SKIPPED (HiChIP does not use linkers)"
```

### Step 2: Genomic Mapping

Maps paired-end reads directly to reference genome using BWA.

```
Input:  sample_R1.fastq.gz, sample_R2.fastq.gz
Output: step2_mapping/mapped.dedup.bedpe
```

**Mapping Statistics:**
- Total read pairs processed
- Successfully mapped pairs
- Unique PETs after deduplication
- Intra-chromosomal (cis) ratio
- Inter-chromosomal (trans) ratio

### Step 3: PET Purifying (Restriction Fragment Filtering)

Removes same-fragment PETs (invalid ligation products) using restriction site information.

```
Input:  step2_mapping/mapped.dedup.bedpe
        restriction_sites.bed
Output: step3_purifying/purified.valid.bedpe
        step3_purifying/purified.sameres.bedpe (removed)
```

**Why This Matters:**
- Same-fragment PETs are artifacts from self-ligation
- Valid PETs span different restriction fragments
- This step is critical for HiChIP data quality

### Step 4: PET Categorization

Classifies PETs based on genomic distance.

```
Input:  step3_purifying/purified.valid.bedpe
Output: step4_categorization/categorized.ipet  (inter-ligation PETs)
        step4_categorization/categorized.spet  (self-ligation PETs)
        step4_categorization/categorized.opet  (other PETs)
```

**PET Categories:**
- **iPET**: Long-range interactions (distance > 1kb for HiChIP)
- **sPET**: Short-range/self-ligation (distance ≤ 1kb)
- **oPET**: Wrong orientation or other issues

### Step 5: Peak Calling

Calls enrichment peaks from self-ligation PETs using MACS3.

```
Input:  step4_categorization/categorized.spet
Output: step5_peaks/peaks_peaks.narrowPeak
        step5_peaks/peaks_summits.bed
        step5_peaks/peaks_peaks.xls
```

**Note**: Peaks represent protein binding sites (e.g., H3K27ac marks).

### Step 6: Loop Calling with Statistical Significance

Identifies significant chromatin loops using distance-corrected statistics.

```
Input:  step4_categorization/categorized.ipet
Output: step6_loops/loops.cluster.filtered
        step6_loops/loops.cluster.FDRfiltered.txt
```

**Sub-steps:**

1. **Pre-clustering (6.1)**
   - Extends each tag by `extension-length` bp
   - Groups overlapping extended tags into clusters

2. **Anchor Clustering (6.2)**
   - Merges pre-clusters into anchor pairs
   - Creates interaction matrix between anchors

3. **Statistical Significance (6.3)**
   - Distance-corrected Poisson test (HiChIP-specific)
   - Benjamini-Hochberg FDR correction
   - Filters by `fdr-cutoff` threshold

## Output Files

```
output_dir/
├── step1_linker_filtering/     (empty - HiChIP skips this)
├── step2_mapping/
│   ├── mapped.dedup.bedpe
│   └── mapping_stats.txt
├── step3_purifying/
│   ├── purified.valid.bedpe
│   └── purified.sameres.bedpe
├── step4_categorization/
│   ├── categorized.ipet
│   ├── categorized.spet
│   └── categorized.opet
├── step5_peaks/
│   ├── peaks_peaks.narrowPeak
│   ├── peaks_summits.bed
│   └── peaks_peaks.xls
└── step6_loops/
    ├── loops.cluster.filtered      (all loops meeting iPET threshold)
    └── loops.cluster.FDRfiltered.txt (significant loops)
```

## Output Format

### Loop File Format

The final loop file (`loops.cluster.FDRfiltered.txt`) contains:

| Column | Description |
|--------|-------------|
| chr1 | Chromosome of anchor 1 |
| start1 | Start position of anchor 1 |
| end1 | End position of anchor 1 |
| chr2 | Chromosome of anchor 2 |
| start2 | Start position of anchor 2 |
| end2 | End position of anchor 2 |
| iPET_count | Number of supporting iPETs |
| expected | Expected count (distance-corrected) |
| distance | Genomic distance between anchors |
| p_value | Raw p-value |
| FDR | Benjamini-Hochberg adjusted p-value |

### Peaks File Format (narrowPeak)

| Column | Description |
|--------|-------------|
| chrom | Chromosome |
| start | Peak start |
| end | Peak end |
| name | Peak name |
| score | Score |
| strand | Strand (.) |
| signalValue | Signal value |
| pValue | -log10(p-value) |
| qValue | -log10(q-value) |
| peak | Summit position relative to start |

## HiChIP vs ChIA-PET

| Feature | HiChIP | ChIA-PET |
|---------|--------|----------|
| Linker sequences | **No** | Yes |
| Restriction enzyme | Yes (MboI/DpnII) | No |
| Input requirement | Low (1M cells) | High (100M cells) |
| Purification step | Fragment-based | Merge-based |
| Statistical model | Distance-corrected | Standard Poisson |

## Quality Metrics

**Good HiChIP Data:**
- Mapping rate: >70%
- Cis ratio: >60%
- Valid PET ratio: >50%
- iPET percentage: >40%
- Peaks called: Depends on protein

**Warning Signs:**
- Low cis ratio (<40%): May indicate ligation issues
- High same-fragment rate (>50%): Incomplete digestion
- Few peaks: Low enrichment or wrong antibody

## Tips

1. **Restriction Sites**: Always use the correct enzyme for your protocol
2. **BWA-MEM**: Recommended for HiChIP (`--use-bwa-mem`)
3. **Distance Correction**: HiChIP uses distance-corrected statistics automatically
4. **Peak Validation**: Compare peaks with ChIP-seq if available
5. **Sequencing Depth**: Aim for >100M read pairs for comprehensive analysis

## Troubleshooting

### High same-fragment rate
- Check restriction enzyme digestion efficiency
- Verify restriction sites file matches your enzyme
- Consider re-digestion or protocol optimization

### No peaks called
- Verify antibody specificity
- Check enrichment with ChIP-seq
- May need more sequencing depth

### Few significant loops
- Increase sequencing depth
- Lower FDR threshold for exploration
- Check if peaks are present (loops anchor at peaks)

### Memory issues
- Reduce threads
- Process chromosomes separately
- Use `--keep-intermediates` to debug step-by-step
