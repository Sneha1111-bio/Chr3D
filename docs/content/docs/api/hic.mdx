# Hi-C Python API

Complete Python API for analyzing Hi-C data for genome-wide chromatin architecture studies.

## Overview

Hi-C captures all chromatin interactions genome-wide without protein-specific enrichment. The Chr3D Hi-C pipeline generates contact matrices at multiple resolutions using pairtools and cooler.

**Note**: Hi-C mode does **not** include peak calling or loop calling (no protein enrichment). Use external tools like HiCCUPS, Mustache, or cooltools for downstream analysis.

## Complete Pipeline Example

```python
import sys
sys.path.insert(0, '/path/to/Chr3D/src')

from chr3d.bulk_hic import HiCPipeline

# Configuration
fastq_r1 = "/data/HiC_sample_R1.fastq.gz"
fastq_r2 = "/data/HiC_sample_R2.fastq.gz"
genome_index = "/ref/hg38.fa"
chrom_sizes = "/ref/hg38.chrom.sizes"
output_dir = "/output/hic_analysis"

# Initialize pipeline
hic_pipeline = HiCPipeline(
    genome_index=genome_index,
    chrom_sizes=chrom_sizes,
    threads=8,
    assembly='hg38',
    min_mapq=30,
    min_distance=1000,
    resolutions=[1000, 5000, 10000, 25000, 50000, 100000]
)

# Run complete pipeline
stats = hic_pipeline.run(
    fastq1=fastq_r1,
    fastq2=fastq_r2,
    output_dir=output_dir,
    sample_id='HiC_experiment',
    cleanup=True  # Remove intermediate files
)

# Print results
print(f"Total pairs: {stats.get('total_pairs', 'N/A'):,}")
print(f"Cis pairs: {stats.get('cis_pairs', 'N/A'):,}")
print(f"Trans pairs: {stats.get('trans_pairs', 'N/A'):,}")
print(f"Output: {output_dir}/HiC_experiment.mcool")
```

## Module Reference

### HiCPipeline

Complete Hi-C processing pipeline using pairtools and cooler.

```python
from chr3d.bulk_hic import HiCPipeline

pipeline = HiCPipeline(
    genome_index: str,              # BWA-indexed genome FASTA
    chrom_sizes: str,               # Chromosome sizes file
    threads: int = 8,               # Number of threads
    assembly: str = 'hg38',         # Genome assembly name
    min_mapq: int = 30,             # Minimum mapping quality
    min_distance: int = 1000,       # Minimum distance for valid pairs
    resolutions: List[int] = [1000, 5000, 10000]  # Matrix resolutions
)
```

#### run()

Run the complete Hi-C pipeline.

```python
stats = pipeline.run(
    fastq1: str,                    # Input R1 FASTQ
    fastq2: str,                    # Input R2 FASTQ
    output_dir: str,                # Output directory
    sample_id: str = 'sample',      # Sample identifier
    cleanup: bool = True            # Remove intermediate files
) -> Dict
```

**Returns:**
```python
{
    'total_pairs': int,
    'mapped_pairs': int,
    'unique_pairs': int,
    'cis_pairs': int,
    'trans_pairs': int,
    'cis_ratio': float,
    'duplicate_rate': float,
    'output_mcool': str
}
```

## Pipeline Steps

The `HiCPipeline.run()` method executes these steps:

### Step 1: BWA-MEM Alignment

Maps paired-end reads to reference genome.

```python
# Internal: Uses subprocess to run BWA-MEM
bwa mem -SP5M -t {threads} {genome_index} {fastq1} {fastq2}
```

### Step 2: Pairtools Parse

Converts aligned BAM to pairs format.

```python
# Internal: pairtools parse
pairtools parse --min-mapq {min_mapq} \
    --walks-policy 5unique \
    --max-inter-align-gap 30 \
    --chroms-path {chrom_sizes} \
    {input_bam}
```

### Step 3: Pairtools Sort

Sorts pairs by genomic coordinates.

```python
# Internal: pairtools sort
pairtools sort --nproc {threads} {input_pairs}
```

### Step 4: Pairtools Dedup

Removes PCR duplicates.

```python
# Internal: pairtools dedup
pairtools dedup --max-mismatch 3 {input_pairs}
```

### Step 5: Pairtools Select

Filters for valid cis pairs.

```python
# Internal: pairtools select
pairtools select '(pair_type == "UU") and (chrom1 == chrom2) and (abs(pos1 - pos2) >= {min_distance})'
```

### Step 6: Cooler Cload

Creates contact matrix from pairs.

```python
# Internal: cooler cload pairs
cooler cload pairs -c1 2 -p1 3 -c2 4 -p2 5 \
    {chrom_sizes}:{resolution} \
    {input_pairs} \
    {output_cool}
```

### Step 7: Cooler Zoomify

Creates multi-resolution .mcool file.

```python
# Internal: cooler zoomify
cooler zoomify --resolutions {resolutions} {input_cool}
```

## Working with Output Files

### Loading .mcool Files

```python
import cooler

# Load specific resolution
clr = cooler.Cooler("sample.mcool::resolutions/10000")

# Get matrix for region
matrix = clr.matrix(balance=False).fetch("chr1:1000000-2000000")

# Get balanced (normalized) matrix
matrix_balanced = clr.matrix(balance=True).fetch("chr1:1000000-2000000")

# Get bins
bins = clr.bins()[:]

# Get chromosome info
chroms = clr.chromsizes
```

### Normalization with Cooler

```python
import cooler

# Load cooler
clr = cooler.Cooler("sample.mcool::resolutions/10000")

# ICE normalization (iterative correction)
cooler.balance_cooler(clr, store=True)

# Access balanced weights
weights = clr.bins()['weight'][:]
```

## Downstream Analysis

### TAD Calling with Cooltools

```python
import cooltools
import cooler

clr = cooler.Cooler("sample.mcool::resolutions/10000")

# Calculate insulation score
insulation = cooltools.insulation(clr, window_bp=[100000, 200000])

# Find TAD boundaries
boundaries = cooltools.find_boundaries(insulation)
```

### Compartment Analysis

```python
import cooltools

clr = cooler.Cooler("sample.mcool::resolutions/100000")

# Calculate eigenvectors for compartments
eigs = cooltools.eigs_cis(clr, n_eigs=3)

# First eigenvector corresponds to A/B compartments
compartments = eigs[0]
```


## QC Statistics

The pipeline generates QC statistics in `qc/sample_pairs.stats`:

| Metric | Description |
|--------|-------------|
| total | Total read pairs |
| total_mapped | Successfully mapped pairs |
| total_dups | Duplicate pairs |
| total_nodups | Unique pairs |
| cis | Same-chromosome pairs |
| trans | Different-chromosome pairs |
| cis_1kb+ | Cis pairs >1kb apart |
| cis_10kb+ | Cis pairs >10kb apart |
| summary/frac_cis | Fraction of cis pairs |

## Quality Metrics

**Good Hi-C Data:**

| Metric | Good | Acceptable | Poor |
|--------|------|------------|------|
| Mapping rate | >90% | 70-90% | <70% |
| Cis ratio | >60% | 40-60% | <40% |
| Cis >10kb | >40% | 20-40% | <20% |
| Duplicate rate | <20% | 20-40% | >40% |

## Resolution Guidelines

| Resolution | Use Case | Memory |
|------------|----------|--------|
| 1kb | Fine-scale loops | High |
| 5kb | Loop detection | Medium |
| 10kb | TAD calling | Medium |
| 25kb | Compartments | Low |
| 50kb | Overview | Low |
| 100kb | Whole-genome view | Very Low |

## Hi-C vs HiChIP vs ChIA-PET

| Feature | Hi-C | HiChIP | ChIA-PET |
|---------|------|--------|----------|
| Enrichment | None | ChIP | ChIP |
| Interactions | All | Protein-specific | Protein-specific |
| Resolution | Genome-wide | Focused | Focused |
| Input cells | 1-10M | 1M | 100M |
| Loop calling | External tools | Built-in | Built-in |
| Peak calling | No | Yes | Yes |
| Contact matrix | Yes (.mcool) | No | No |

## Generating Chromosome Sizes

```python
# Using pysam
import pysam

fasta = pysam.FastaFile("/ref/hg38.fa")
with open("/ref/hg38.chrom.sizes", "w") as f:
    for chrom in fasta.references:
        f.write(f"{chrom}\t{fasta.get_reference_length(chrom)}\n")
```

Or using samtools:
```bash
samtools faidx /ref/hg38.fa
cut -f1,2 /ref/hg38.fa.fai > /ref/hg38.chrom.sizes
```

## Tips

1. **Sequencing Depth**: Aim for >500M read pairs for 1kb resolution
2. **Memory**: High resolutions require significant RAM (>64GB for 1kb)
3. **Storage**: .mcool files can be large (10-100GB)
4. **Normalization**: Always use balanced matrices for analysis
5. **Multiple Resolutions**: Generate multiple for different analyses
